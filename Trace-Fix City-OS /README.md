# CityOS Integration Roadmap

## Overview

TraceFix is responsible for **planning, verification, and synthesis** of multi-agent coordination protocols.

CityOS is responsible for **runtime execution, lifecycle management, communication, monitoring, permissions, and resource access**.

The boundary between the two systems is a **verified application package**.

```
TeLLMe Intent
      ↓
TraceFix Planner
      ↓
IR Generation
      ↓
PlusCal / TLA+ Verification
      ↓
Counterexample-Guided Repair
      ↓
Verified Workspace
      ↓
Application Artifact Synthesis
      ↓
CityOS Module Package
      ↓
CityOS Runtime
      ↓
Agent Execution + Monitoring
```

## Responsibilities

### TraceFix

TraceFix is responsible for:

* Generating the Intermediate Representation (IR)
* Producing PlusCal and TLA+ specifications
* Running TLC verification
* Repairing protocol violations using counterexamples
* Extracting verified state machines
* Generating agent prompts and runtime artifacts
* Packaging verified workspaces for deployment

TraceFix is **not responsible for runtime orchestration after deployment**.

### CityOS

CityOS acts as the runtime operating system.

CityOS is responsible for:

* Loading application modules
* Dependency resolution
* Lifecycle management
* Resource allocation
* Stream access control
* Privacy enforcement
* Runtime monitoring
* Inter-module communication
* ConcordFS integration
* Failure handling and recovery

CityOS determines which runtime libraries, services, and communication mechanisms are attached to a deployed application.

## Deployment Model

The output of TraceFix is a verified workspace containing:

```
workspace/
├── description.md
├── tools.json
├── spec/
│   ├── ir.json
│   ├── Protocol.tla
│   ├── states.json
│   └── summary.json
├── prompts/
│   ├── runtime_a/
│   └── runtime_b/
└── output/
```

The workspace is then transformed into a CityOS application package.

A package may contain:

```
cityos-module/
├── workspace/
├── manifest.json
├── Dockerfile
├── entrypoint.sh
├── requirements.txt
└── metadata/
```

The package contains all assets necessary for CityOS to instantiate the application.

## Agent Execution Model

Agents are generated by TraceFix but executed by CityOS.

The package contains:

* Agent prompts
* Agent configuration
* State machine artifacts
* Coordination metadata
* Dependency declarations

CityOS loads these artifacts and instantiates runtime agents under CityOS supervision.

## ConcordFS

ConcordFS serves as the communication substrate between deployed modules.

TraceFix does not implement ConcordFS.

Instead, TraceFix declares:

* Required communication channels
* Message schemas
* Coordination constraints
* State-machine requirements

CityOS maps those requirements onto its ConcordFS runtime implementation.

## Future Work

Planned integration work includes:

* Automated CityOS package synthesis
* Module manifest generation
* Dependency declaration generation
* ConcordFS channel declarations
* Runtime topology monitoring
* Dynamic deployment and scaling of verified agent systems

The long-term goal is for TraceFix to emit a fully verified CityOS-ready application package that can be deployed directly into the CityOS runtime environment.
